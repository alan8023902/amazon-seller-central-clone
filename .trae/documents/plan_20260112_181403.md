# Global Snapshot 重构计划

## 一、修改目标
严格按照参考截图实现Global Snapshot，包含A区（左侧合并块）和B区（右侧8个格子）的完整结构和内容。

## 二、实现方案

### 1. 结构设计
- **外层容器**：使用单个卡片容器，圆角4px，边框#D5D9D9，无阴影
- **A区**：左侧合并块，包含销售额和未解决订单两个单元，用1px竖分割线隔开
- **B区**：右侧格子区，4列×2行布局，8个格子，用1px分割线隔开

### 2. 内容实现

#### A区左单元（销售额）
- 标题：销售额 + 右上 caret
- 大数字：US$49.95
- 小字：今天到目前为止
- 下方折线图（含纵轴刻度100/50/0，底部日期：1月1日/4日/7日）

#### A区右单元（未解决的订单）
- 标题：未解决的订单 + 右上 caret
- 大数字：6
- 小字：总计
- 明细三行（左文案 + 右侧数字对齐）：
  - FBM 未发货 0
  - FBM 待处理 0
  - 亚马逊物流待处理 6

#### B区8个格子
**第一行(左->右)**：
1. 买家消息(0, 需要注意的问题)
2. 推荐报价百分比(100%, 2天前)
3. 卖家反馈(星星+5.00, 过去一年(2))
4. 付款(US$228.31, 总余额)

**第二行(左->右)**：
5. 库存绩效指标(400, 当前)
6. 全球促销销售额(没有可用数据 + 链接"了解更多信息")
7. 广告销售(US$0.00, 今天到目前为止)
8. 广告曝光量(0, 今天到目前为止)

### 3. 视觉规范
- 外层卡片圆角4px，内部格子无圆角
- 外边框#D5D9D9，内部所有分割线#E7EAEA
- 标题和关键数字颜色#002E35，说明文字#565959，链接#007185
- 卡片内边距12-16px，整体紧凑

### 4. 技术实现
- 使用Tailwind CSS实现布局和样式
- 复用现有的StarRating和TinySparkline组件
- 添加必要的CSS辅助类到index.css
